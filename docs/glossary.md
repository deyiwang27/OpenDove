# Glossary

## OpenDove

The overall software delivery system and control layer.

## AI-Native Development

A workflow where AI tools are treated as default participants in software delivery rather than optional assistants at the edge.

## Coding Assistant

A code-focused execution tool such as Claude Code, Codex, or Gemini that can edit code, run tools, and solve implementation-level tasks.

## Product Manager

The role responsible for defining what should be built, why it matters, and how success is measured.

## Project Manager

The role responsible for execution flow, task assignment, iteration limits, and escalation timing.

## Lead Architect

The role responsible for defining the implementation approach and validating system-level integration.

## Developer

The execution role responsible for implementing changes and producing test evidence.

## AVA

Alignment and Validation Agent. The blocking gate that approves, rejects, or escalates based on scope, intent, and evidence.

## Alignment

Faithfulness between the original user intent and the delivered result.

## Validation

The explicit process that determines whether work satisfies the task contract and should be considered done.

## Escalation

The act of handing a task back to humans or upstream decision-makers when ambiguity, risk, or policy limits prevent safe autonomous continuation.

## Autonomous Software Factory

The long-term OpenDove vision in which users provide goals and boundaries while the system manages most of the software delivery lifecycle with minimal human intervention.
